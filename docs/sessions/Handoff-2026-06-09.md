### Key Discoveries
- **PowerShell pipe corrupts `authorized_keys`.** `Get-Content key.pub | ssh … "cat >> authorized_keys"` appends CRLF; the `\r` breaks the key line. Install keys via inline variable expansion (`printf '%s\n' '$pub'`) or from inside an interactive session instead.
- **Windows OpenSSH enforces ACL strict modes** on `~/.ssh/config` — files created by other tools inherit ACLs (including orphaned SIDs from old accounts) that ssh rejects with "Bad owner or permissions."
- **The capital-K trap recurred.** Jacob typed `cypherKlein` again (same as S7's hour-long diagnosis): invalid user → no `authorized_keys` lookup possible AND passwords check against a nonexistent account, so both auth methods fail confusingly. Fixed structurally: the alias supplies the username, so it can no longer be mistyped.
- **Auth-log forensics:** on Bookworm use `journalctl -u ssh` (the `-t sshd` tag returns nothing). The log proved Jacob's password was typed correctly once (19:31:58, candidate #5) before being reset.
- **The GitHub repo is public** — the Pi pulls anonymously over HTTPS; no deploy key or PAT needed.
